## Tips for Writing

### For Core Functionality
- Focus on what problem it solves
- Include concrete user scenarios
- Show before/after comparisons
- Explain key benefits upfront

### For Customization
- Document all available options clearly
- Provide multiple real-world examples
- Show how to apply changes
- Include best practices

### For Performance
- Include performance metrics and benchmarks
- Explain the optimization strategy
- Show before/after performance impact
- Document any trade-offs

### For Accessibility
- Explain who benefits from this feature
- Document specific accessibility support
- Provide usage examples
- Link to accessibility guidelines

### For User Experience
- Focus on how users feel when using it
- Show workflow improvements
- Include user scenarios
- Highlight integration points

### For Extended Functionality
- Explain how it enhances the core feature
- Show optional configurations
- Document when to use this feature
- Relate back to core functionality
